\begin{abstract}
Remy~\cite{winstein2013tcp} showed that computer-generated congestion control
algorithms can outperform human designs within their training distribution, but
generalize poorly outside it. A natural question is whether alternative
representations---neural networks or structured source code---generalize
better than Remy's decision trees when trained on the same environments with
the same observation and action spaces.

We revisit this setup with three policy representations: Remy's WhiskerTrees,
PPO neural networks, and AlphaCC, an LLM-guided code evolution loop that emits
Python CCAs. In the preserved CS244C artifact, Remy's 1$\times$ tree remains
the strongest single-point method across most of the 9 canonical link rates,
PPO underperforms it at every tested rate, and the retained single-point
AlphaCC run is interpretable but brittle, degrading sharply outside its seed's
comfort zone. The strongest preserved multi-point AlphaCC run improves the
9-point mean from $-$3.02 to $-$0.76, but the broader set of archived
multi-point runs remains high-variance and cross-simulator comparison still
limits causal claims.
The main takeaway is not that LLM evolution beats Remy; it is that readable
code can rediscover TCP-like structure, yet representation alone does not
overcome narrow training or simulator mismatch.
\end{abstract}
